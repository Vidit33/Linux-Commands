## Navigation

```bash
# List files in current directory
ls

# Change into a directory
cd dir_name

# Go back to the home directory
cd

# Stay in the current directory
cd .

# Go one directory up
cd ..

# View a command's manual
man cmd
# Example: man ls
```

## File Operations

```bash
# Create an empty file
touch filename.ext

# Delete a file
rm filename.ext

# Delete a directory (recursive)
rm -r dir_name

# Create a new directory
mkdir dir_name

# Move a file to another directory
mv filename.ext dest_directory/

# Copy a file to another directory
cp filename.ext dest_directory/

# Copy files from Raspberry Pi to your PC over the network
rsync -P pi@<pi-ip>:~/path/to/folder ~/local/path/
```

## File Content & Search

```bash
# Print the contents of a file
cat filename.ext

# Search for a string in output
grep "search_term"

# Pipe the output of one command into another
# Example: list files and filter by name
ls -l | grep "search_term"

# Find the location of a command or binary
which cmd
whereis cmd
```

## Text & Output

## SSH & GitHub Setup

### 1. Generate an SSH key

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

### 2. Add the public key to GitHub

```bash
cd ~/.ssh
cat id_ed25519.pub
# Copy the output and paste it into GitHub → Settings → SSH and GPG keys
```

### 3. Clone a repository using SSH

```bash
git clone git@github.com:Vidit33/Linux-Commands.git
```

### 4. Initialize Git in an existing directory

```bash
git init
git remote add origin git@github.com:Vidit33/Linux-Commands.git
```

### 5. Verify GitHub recognizes your key

```bash
ssh -T git@github.com
```

## Git Workflow

```bash
# Stage all changes
git add .

# Commit with a message
git commit -m "your commit message"

# Push changes to remote
git push

# Pull latest changes
git pull

# View commit history
git log

# Check current status
git status

# Append a small change to a tracked file without opening it
echo "Added new command: top" >> path/to/file.txt

# Show uncommitted changes
git diff
```

### Branches

```bash
# Create a new branch
git branch feature

#List a new branch
git branch(Lists local branches)

#List Local and Remote branches
git branch -a

# Create and switch to a new branch
git switch -c feature

# Push a new branch to remote and set upstream
git push -u origin feature

##Delete a new branch that you created
#Branch is Local
git branch -d 'branch_name'
#branch also exits in github
git push origin --delete feature

```

### Troubleshooting: Switching remote from HTTPS to SSH

```bash
# Check current remote URL
git remote -v

# Switch remote URL to SSH
git remote set-url origin git@github.com:Vidit33/linux_commands.git

# Verify the change
git remote -v
```

If still not working, remove and re-add the remote:

```bash
git remote remove origin
git remote add origin git@github.com:Vidit33/Linux-Commands.git
git remote -v
```

If a global HTTPS override is interfering:

```bash
# Check for the override
git config --global --get url."https://github.com/".insteadOf

# Remove it
git config --global --unset url."https://github.com/".insteadOf

# Re-add the SSH remote
git remote remove origin
git remote add origin git@github.com:Vidit33/Linux-Commands.git
```

## Raspberry Pi

### Reset a Forgotten Password

1. Remove the SD card and attach it to your PC.
2. Open `cmdline.txt` on the SD card and append `init=/bin/sh` after the existing text.
3. Eject the SD card, reinsert it into the Pi, and power on — it boots into a root shell.
4. Run the following commands:

```bash
mount -o remount,rw /   # remount root filesystem as read-write
passwd pi               # set new password
sync
exec /sbin/init         # reboot
```

5. After rebooting, **remove** `init=/bin/sh` from `cmdline.txt`, otherwise the Pi will always boot into root shell.

### Check Pi Health

```bash
vcgencmd measure_volts        # measure supply voltage
vcgencmd measure_clock arm    # measure ARM CPU clock speed
```

### Enable NetworkManager

```bash
sudo systemctl enable NetworkManager
sudo systemctl start NetworkManager
```
